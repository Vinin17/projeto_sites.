Questões sobre HTML5 (Estrutura e Semântica)

1. As tags header, main e footer deixam claro o que cada parte deve estar quando o programa for executado. Já o div é como se fosse uma caixa vazia, naqual apenas se adicona coisas sem uma ordem.
Logo, se for substituir as outras tags por div, os codigos estarão "bagunçados" ou pelo menos sem uma estrutura para à organiza-la

2. Se a imagem for apenas decorativa, o atributo deve ser: alt="" (vazio), isso diz ao leitor de tela "ignore essa imagem"

3. No celular abre um teclado adaptado, com @ e .com, facilitando a digitação. Alem de garantir ama validação adequada, pois o navegador já verifica se o texto parece com um email.
Coisa na qual o type="text" não proporciona

4. Ao usar uma ol, a ordem importa, então deve-se usar em passo a passo, receita, entre outros.
Já ao usar um ul, a ordem não tem importancia, então pode usar em listas de compra, itens em menu, entre outros.

Questões sobre Git e GitHub (Controle de Versão)

5. Ao commitar sem o add, o git ignora as mudanças feitas e nada novo será salvo

6.
git init: cria um novo repositório do zero, usado quando está começando um novo projeto
git clone: copia um repositório já existente, usado quando quer trabalhar em um projeto já criado

7. O desenvolvedor, pois o git só mostra onde está o conflito. Você que tem que decidi qual versão manter, pois o git não sabe qual é a versão correta

8. é a junção de git fetch: baixa as atualizações do repositório remoto, e do git merge: junta as atualizações com seu codigo
